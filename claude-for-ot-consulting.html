Pull Request automatiche su GitHub

Claude Code si integra nativamente con GitHub: crea PR, revisiona codice con agenti multi-livello, e risponde a menzioni @claude direttamente nelle issue e PR.

PR da conversazione

"Implementa la feature JIRA-123 e apri una PR" — Claude scrive il codice, crea il branch, e apre la Pull Request con descrizione strutturata.

Code Review multi-agente

Una flotta di agenti specializzati analizza le modifiche nel contesto dell'intero codebase. Trova bug, vulnerabilità e regressioni con severity tagging.

@claude nelle PR e Issue

Menziona @claude in qualsiasi commento GitHub per chiedere fix, implementazioni o spiegazioni. Rispetta le regole in CLAUDE.md.

# Setup GitHub Action - uses: anthropics/claude-code-action@v1 with: anthropic_api_key: ${{ secrets.ANTHROPIC_API_KEY }} # Poi, in qualsiasi PR o Issue: @claude fix the TypeError in the dashboard @claude review this PR # → Claude analizza, implementa, commenta inline

Insegna a Claude le regole del tuo team

Le Skills sono moduli riutilizzabili che codificano conoscenza di dominio, workflow e best practice. Si attivano automaticamente quando servono, si condividono via Git.

Struttura modulare

Ogni Skill è una directory con SKILL.md + script + template. Attivazione automatica basata sul contesto — nessun invoke manuale.

Gerarchia enterprise

4 livelli: Managed → User → Project → Local. Le policy aziendali hanno sempre la priorità. Configurabili dal team IT centrale.

Versionati in Git

Le Skills vivono in .claude/skills/ nel repo. Ogni dev ha gli stessi standard. Cambio? Un commit e tutti sono allineati.

1M di contesto, senza perdere il filo

Il context rot è il degrado delle prestazioni all'aumentare del contesto. A 1M di token, Opus 4.6 mantiene il 78.3% di accuratezza — GPT-5.4 crolla al 36.6%, Gemini 3.1 Pro al 25.9%.

Retrieval a 1M token

Sul benchmark MRCR v2 (8-needle), Opus 4.6 recupera informazioni distribuite in tutto il contesto con il 78.3% di match ratio — il doppio di GPT-5.4.

Il problema dei competitor

GPT-5.4 parte da ~80% a 128K ma crolla a 36.6% a 1M. Gemini 3.1 Pro scende a 25.9%. Sonnet 4.5 è inutilizzabile (18.5%).

Impatto pratico

Codebase grandi, log estesi, documentazione complessa — Claude mantiene coerenza dove gli altri "dimenticano" le informazioni iniziali.

Benchmark: MRCR v2, 8-needle. Gemini misurato da Context Arena sullo stesso benchmark, non self-report Google. Score OpenAI è media 128K–256K.
