Day 1 — Parte teorica

Cos'è Claude e come usarlo

Claude: i fondamenti (20 min)

Differenza tra Claude.ai, API e Claude Code. Modelli disponibili (Sonnet, Opus, Haiku) e quando usarli. Context window e system prompt.

Claude Code in azione (30 min)

Demo live: da zero a progetto funzionante in 10 minuti. Comandi essenziali, CLAUDE.md, prompt engineering per sviluppatori.

MCP, Skills & Plugin (20 min)

Model Context Protocol: collegare tool esterni (GitHub, Jira, DB). Skills: moduli riutilizzabili con standard e workflow del team. Plugin: estensioni installabili con un comando (Playwright, Context7, Serena).

Concetti chiave

Context Window System Prompt CLAUDE.md /init /compact /model MCP Server Skills Plugin Prompt Engineering

Regola d'oro: più contesto dai a Claude, migliore sarà il risultato. Il CLAUDE.md è il tuo alleato.

Il cuore del corso

Un progetto reale, costruito insieme

Ogni team riceve un'area tematica e ha fino al D3 per costruire un progetto funzionante con Claude Code — dall'idea alla demo live.

Team da ~3 persone

Formati eterogenei per competenze. Ogni gruppo ha un tutor di supporto dedicato.

Stack libero

Frontend, backend, fullstack — il team sceglie le tecnologie. Claude Code supporta qualsiasi linguaggio.

100% con Claude Code

Dall'init del progetto alla PR finale. Skills, Plugin e MCP inclusi nel workflow.

Demo Day finale

Ogni team presenta il progetto: demo live, come ha usato Claude, lezioni apprese.

Obiettivo: non è il codice perfetto, ma imparare a collaborare con l'AI. Il progetto è il veicolo — le competenze acquisite sono il vero output.

Workflow del progetto

Da zero a demo in 3 step

Il percorso di ogni team segue 3 fasi chiare, ognuna supportata dal tutor.

STEP 1 — D1

Ideazione & Setup

  • Sorteggio area tematica
  • Brainstorming con il team
  • Definizione scope (cosa fa, cosa no)
  • claude /init + CLAUDE.md
  • Validazione del piano con il tutor
STEP 2 — D2

Sviluppo intensivo

  • Coding assistito con Claude Code
  • Integrazione MCP server e Plugin
  • Creazione Skills per il progetto
  • Checkpoint a metà giornata
  • Riduzione scope se necessario
STEP 3 — D3

Sprint finale & Demo

  • Fix e polish dell'ultimo miglio
  • Preparazione presentazione
  • Demo live (5-7 min per team)
  • Racconto dell'uso di Claude
  • Q&A e votazione miglior progetto

Valutazione

Questionari PRE & POST corso

Due questionari per misurare l'impatto della formazione e raccogliere feedback strutturato.

PRE — Prima del corso

  • Valutazione competenze attuali dello studente
  • Suggerimenti su aree tematiche per i progetti
  • Aspettative e obiettivi personali sul corso

POST — Dopo il corso

  • Valutazione competenze acquisite (stesso formato del PRE per confronto)
  • Feedback sul corso: contenuti, ritmo, supporto
  • Spunti e suggerimenti per migliorare le prossime edizioni

Perché due questionari? Il confronto PRE vs POST permette di misurare oggettivamente il progresso di ogni partecipante e l'efficacia complessiva della formazione.
