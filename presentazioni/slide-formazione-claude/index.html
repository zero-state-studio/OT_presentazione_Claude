Introduzione

Cos'è Claude Code

Un agente di coding direttamente nel terminale, pensato per sviluppatori.

CLI nativo

Si lancia dal terminale con claude. Legge, scrive e modifica file direttamente nel tuo progetto senza bisogno di IDE.

Agentico

Esplora il codebase, esegue comandi, crea branch, apre PR e gestisce task multi-step in autonomia.

Sicuro by design

Sistema di permessi granulare: chiede conferma prima di operazioni rischiose. Tre livelli: suggest, auto-edit, full-auto.

Multi-piattaforma

Disponibile come CLI, desktop app (Mac/Win), web app (claude.ai/code) e estensioni IDE (VS Code, JetBrains).

Estensioni

Model Context Protocol

Lo standard aperto per connettere Claude a qualsiasi tool, database o servizio esterno.

Cos'è MCP

Protocollo client-server che espone tools, resources e prompts a Claude. Basato su JSON-RPC 2.0 over stdio o HTTP/SSE.

Server popolari

GitHub, Slack, Linear, Notion, Postgres, Playwright, Sentry, filesystem e centinaia di community server.

Configurazione

Definisci i server in .claude/settings.json sotto la chiave mcpServers. Supporta env vars e args personalizzati.

Creare un server MCP

SDK ufficiali in Python e TypeScript. Esponi funzioni come tools e Claude le chiamera in autonomia durante il task.

Personalizzazione

Skills & Hooks

Estendi il comportamento di Claude Code con automazioni e competenze riutilizzabili.

Skills

Prompt riutilizzabili in .claude/skills/ che insegnano a Claude competenze specifiche. Si invocano con /nome-skill e possono includere checklist, template e workflow.

Hooks

Comandi shell eseguiti automaticamente su eventi: pre/post tool call, notifiche, stop. Configurati in settings.json sotto la chiave hooks.

CLAUDE.md

File markdown alla root del progetto. Claude lo legge all'avvio di ogni sessione. Ideale per: stack tecnologico, convenzioni di codice, comandi di build/test.

Subagent

Claude Code puo lanciare agenti paralleli per task indipendenti (ricerca, test, code review) mantenendo il contesto nel thread principale.
